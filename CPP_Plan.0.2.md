### 2. 🧹Фильтрация (Очистка)
Программа отбрасывает очевидный мусор (например, если пульс внезапно стал 0 на одну секунду, а потом снова 70 — это отошел провод, а не остановка сердца). Медицинские программы "глупые": если провод отошел и вольтаж прыгнул до небес, они кричат: «Аритмия! Ужас!». Но сердце физически не может сокращаться со скоростью 600 ударов в минуту или мгновенно остановиться на одну секунду, а в следующую выдать пульс 70.
«Ложные тревоги» (Как убрать мусор без высшей математики) - решение на C++: написать алгоритм валидации артефактов (физических помех).

##### Медицинские и физиологические фильтры (простая логика и условия if/else):

Классические методы обработки сигналов в контексте электрокардиографии (ЭКГ) --- это набор детерминированных математических подходов, которые фильтруют, преобразуют и анализируют график сердечного ритма для поиска конкретных событий (например, начала сокращения желудочков) без использования искусственного интеллекта и нейросетей. 

* * * * *

Основные компоненты классических алгоритмов

1\. Фильтрация (Filtering)

Это процесс очистки сырого сигнала от физиологических и аппаратных помех.

-   Сетевая наводка: Удаляется режекторными фильтрами на частоте \(50\) Гц (в Европе и России) или \(60\) Гц.
-   Дрейф изолинии (дыхание пациента): Убирается с помощью фильтров высоких частот (ФВЧ), например, с порогом \(0.5\) Гц.
-   Высокочастотный мышечный шум: Удаляется фильтрами низких частот (ФНЧ) с порогом около \(10-15\) Гц.
-   *Результат:* Исходный сигнал становится ровным, с ярко выраженными зубцами. 
2\. Вейвлет-преобразования (Wavelet Transforms)

Математический метод, который раскладывает сложный сигнал на составляющие «волны» разного масштаба (частотно-временной анализ). 

-   В отличие от обычного преобразования Фурье, вейвлет позволяет не только узнать, какие частоты есть в сигнале, но и в какой именно момент времени они произошли. 
-   Это мощный инструмент для подавления шума и одновременного извлечения полезных, резких всплесков (таких как QRS-комплекс) на фоне более сглаженных зубцов P и T. 

3\. Детекторы QRS-комплексов (QRS Detectors)

Программы или цифровые устройства, которые автоматически определяют моменты сердцебиения. 

-   Алгоритм Пан-Томпкинса (Pan-Tompkins): «Золотой стандарт» классических алгоритмов. Он берет отфильтрованный сигнал, пропускает его через производную (для определения крутизны зубцов), возводит сигнал в квадрат (для усиления резких пиков), сглаживает скользящим окном и применяет систему динамических порогов. Пик этого обработанного сигнала соответствует R-зубцу сердца.
-   *Зачем это нужно:* База для любого дальнейшего анализа (например, вычисления ЧСС или вариабельности сердечного ритма). 

* * * * *

Где и почему это применяется?

-   Экономичность: Классические алгоритмы не требуют больших вычислительных мощностей или графических процессоров (GPU).
-   Мобильность: Они работают на обычных микроконтроллерах (в портативных холтерах, умных часах, простейших кардиографах) благодаря низкому потреблению энергии и малому использованию памяти.



##### Что должна делать программа (Три простых правила)


Нужно написать логику, которая проверяет пульс каждую секунду. Переведем слова врачей на язык кода:

   1. Правило «Всё хорошо» (Норма):
   * Что говорит врач: «Синусовый ритм в пределах возрастной нормы».
   * Что это в коде: Если пульс находится в диапазоне от 50 до 140, программа считает, что всё в порядке.
   
   
   2. Правило «Провод отошел» (Помеха/Шум):
   * Что говорит врач: «Артефакты записи, связанные с физической активностью».
   * Что это в коде: Сердце человека физически не может выдать пульс 400 или 500 ударов. Если в файле появилось число больше 200, это не болезнь — это задел провод. Программа должна пометить эту строчку как ШУМ.
   
   
   3. Правило «Остановка сердца» (Пауза ритма):
   * Что говорит врач: «Эпизод асистолии / синоатриальной блокады».
   * Что это в коде: Если пульс внезапно равен 0 или меньше 30 ударов в минуту — это опасный момент. Программа должна пометить эту строчку как КРИТИЧЕСКАЯ ПАУЗА.

Как это выглядит в коде:

   * Фильтр «Отошедший провод» (Дрейф изолинии): Если датчик отклеился, значения напряжения улетают в космос. Вы пишете условие: «Если значение напряжения $> X$ милливольт на протяжении более 0.5 секунд, то этот кусок данных мы не считаем аритмией. Мы помечаем его тегом [ПОМЕХА: ОТОШЕЛ ДАТЧИК] и не показываем врачу как тревогу».
   
   
   * Фильтр «Физиологический предел»: Если между двумя ударами сердца прошло менее 0.2 секунд (что соответствует пульсу > 300 уд/мин, что невозможно для живого человека в покое) — это механический шум (человек почесал грудь, задел провод рукой). Код: помечаем этот пик как [ШУМ] и сглаживаем его.
   
   
   * Фильтр «Контекстный анализ»: Если программа видит «остановку сердца» на 2 секунды, ваш код должен проверить, что было до и после. Если до этого пульс был стабилен (70-70-70-0-70-70), то это технический сбой (потеря пакета данных). Если бы это была реальная блокада сердца, пульс бы плавно замедлялся или менялся.

##### ⁉️ Точка роста (Что делать надо): 
Классический алгоритм Пана-Томпкинса часто ошибается на «грязных» суточных записях Холтера (когда пациент идет, и электроды трутся об одежду). Нужно взять готовый сишный код и дописать адаптивные пороги фильтрации (логику «умного» отсечения шумов), создав свою модификацию (условно, Pan-Tompkins++).

Вместо того чтобы пытаться идеально отфильтровать 100% шума (что физически невозможно, если провод оторвался), нужно научить свой алгоритм не фильтровать, а распознавать безнадежный шум.
Это называется SQA (Signal Quality Assessment — Оценка качества сигнала):

   1. Считаем среднюю амплитуду (высоту) нормального ЭКГ-сигнала.
   
   
   2. Если в какой-то секунде Пана-Томпкинса начинает «колбасить» и амплитуда прыгает в 5–10 раз выше нормы (пациент почесал грудь) — ваш алгоритм на C++ не пытается это очистить.
   
   
   3. Код просто ставит метку: Участок с 14:02:10 по 14:02:15 — ШУМ, данные недостоверны.
   
   
   4. Врач открывает программу, видит серую полосу вместо графиков и говорит вам спасибо. Почему? Потому что вы не подсунули ему 50 ложных аритмий на этом участке, а честно сказали: «Здесь больной шевелился, пропусти этот отрезок».


Результат для врача: программа отфильтрует 90% этого мусора. Вместо 1000 ложных тревог врач откроет отчет и увидит всего 5–10 реальных аномалий, которые действительно требуют его внимания.

Пишем базовый код на C++


Ниже представлен простой, но рабочий код на C++. Он делает три вещи: открывает этот файл, считывает данные в структуру (память) и применяет те самые правила фильтрации if/else, о которых мы говорили.

    #include <iostream>#include <fstream>#include <vector>
    // 1. Создаем структуру для хранения одной строчки данныхstruct HolterPoint {
        double time;
        int pulse;
        std::string status; // Сюда запишем вывод: "Норма", "Шум" или "Пауза"
    };
    int main() {
        // 2. Открываем файл для чтения
        std::ifstream file("holter.txt");
        if (!file.is_open()) {
            std::cout << "Ошибка: Не удалось открыть файл holter.txt" << std::endl;
            return 1;
        }
    
        std::vector<HolterPoint> data;
        double t;
        int p;
    
        // 3. Читаем файл построчно до самого конца (Парсер)
        while (file >> t >> p) {
            HolterPoint point;
            point.time = t;
            point.pulse = p;
           
            // 4. Логика фильтрации и анализа (Умный движок)
            if (p > 200) {
                point.status = "ШУМ (Проверить датчик)";
            } else if (p == 0) {
                point.status = "КРИТИЧЕСКАЯ ПАУЗА РИТМА";
            } else {
                point.status = "Норма";
            }
    
            // Сохраняем обработанную точку в наш массив (вектор)
            data.push_back(point);
        }
        file.close();
    
        // 5. Выводим результат на экран (Имитация отчета для врача)
        std::cout << "=== ОТЧЕТ ДЛЯ ВРАЧА ===" << std::endl;
        for (const auto& point : data) {
            std::cout << "Время: " << point.time << " сек | "
                      << "Пульс: " << point.pulse << " | "
                      << "Статус: " << point.status << std::endl;
        }
    
        return 0;
    }

------------------------------
 Как это работает внутри компьютера?

 

   1. std::ifstream открывает текстовый файл на жестком диске.
   2. Цикл while (file >> t >> p) бежит по файлу. На каждом шаге он берет первую цифру (время) и кладет в переменную t, а вторую цифру (пульс) — в p.
   3. Конструкция if/else мгновенно классифицирует эту секунду. Она не путает помеху (pulse > 200) с реальной остановкой сердца (pulse == 0).
   4. В конце программа выводит чистый отчет, где проблемные места уже размечены для врача.
