###  Мышечный шум 

Мышечный шум (в медицине его называют электромиографический артефакт или тремор) — это второй главный враг чистого графика ЭКГ.
Если с дрейфом изолинии (медленными «горами» от дыхания) наш медианный фильтр справляется на ура, то против мышечного шума он бессилен. 
Тут нужна совершенно другая математика


------------------------------
## Как работает мышечный шум (Простыми словами)
Сердце — это мышца, которая сжимается по команде электрического импульса. Прибор Холтера ловит это электричество через электроды на коже.
Но вокруг сердца есть и другие мышцы: грудные, межреберные, мышцы рук и спины. Когда пациент замерз и дрожит, идет, берет в руки тяжелую сумку или просто нервничает, его скелетные мышцы тоже начинают сокращаться и генерировать микротоки.
Для прибора Холтера это выглядит как хаотичная, бешеная «пила» из мелких острых зубчиков, которая накладывается поверх кардиограммы.
В чем главная опасность:
Эти мелкие мышечные зубчики по своей частоте и форме очень похожи на опасную патологию — фибрилляцию предсердий (мерцательную аритмию). Старый софт вроде «Миокарда» постоянно путает тремор с аритмией, из-за врач и сидит над лентами часами, перепроверяя всё глазами

.
------------------------------
## Как убрать мышечный шум?

Поскольку мышечный шум — это очень быстрые и частые колебания (высокая частота), а полезные волны ЭКГ более плавные (низкая частота), нам нужен Фильтр Низких Частот (ФНЦ) или Low-Pass Filter. Он пропускает медленные полезные сигналы, но полностью срезает и сглаживает быструю мышечную «пилу».
Простым решением будет Экспоненциальное скользящее среднее (EMA — Exponential Moving Average). Это невероятно быстрый алгоритм, который пишется всего в одну строчку кода и не требует сложной математики с синусами.
Логика алгоритма: мы бежим по графику и говорим: «Новая чистая точка — это смесь на 90% из предыдущей чистой точки и всего на 10% из текущей грязной». За счет этого все резкие мышечные прыжки мгновенно сглаживаются.


## Пример фильтра против дрожи на C++

      #include <iostream>#include <vector>
      // Фильтр низких частот (EMA) для удаления мышечного шумаstd::vector<double> removeMuscleNoise(const std::vector<double>& dirtySignal, double alpha) {
          std::vector<double> cleanSignal(dirtySignal.size());
         
          // Стартовая точка остается такой же
          cleanSignal[0] = dirtySignal[0];
         
          // Бежим по графику со второй точки
          for (size_t i = 1; i < dirtySignal.size(); ++i) {
              // Формула сглаживания: alpha — это коэффициент жесткости фильтра (от 0 до 1)
              cleanSignal[i] = alpha * dirtySignal[i] + (1.0 - alpha) * cleanSignal[i - 1];
          }
         
          return cleanSignal;
      }
      int main() {
          // Симулируем сигнал: базовое значение 100, но из-за дрожи мышц
          // график хаотично прыгает туда-сюда: 105, 95, 103, 97...
          std::vector<double> tremorSignal = {100, 105, 95, 103, 97, 104, 96, 102, 98};
         
          // Запускаем фильтр. alpha = 0.2 означает сильное сглаживание шума.
          std::vector<double> cleanSignal = removeMuscleNoise(tremorSignal, 0.2);
         
          std::cout << "Грязная дрожь -> Сглаженный чистый график:\n";
          for (size_t i = 0; i < cleanSignal.size(); ++i) {
              std::cout << tremorSignal[i] << " \t-> \t" << cleanSignal[i] << "\n";
          }
         
          return 0;
      }

## Как это будет работать в стартапе (Конвейер очистки)
В реальности вы объедините оба фильтра в один мощный конвейер (Pipeline) на C++:

   1. Шаг 1: Грязный бинарный файл с флешки Холтера попадает в ваш парсер.
   2. Шаг 2: Сигнал проходит через Медианный фильтр (который вы уже написали на прошлых шагах). Он убирает дрейф изолинии («горы» от вдохов).
   3. Шаг 3: Сигнал сразу же передается в EMA-фильтр (код выше). Он срезает мышечную дрожь и тремор.
   4. Шаг 4: Идеально чистый, ровный график, где остались только настоящие удары сердца, передается в ИИ-модель для анализа.

С таким двухступенчатым конвейером ваша нейросеть будет выдавать потрясающую точность, потому что ей на вход пойдет «стерильный» сигнал без мусора.

## Шаг 1. Математический расчет alpha под реальный Холтер
В профессиональной разработке коэффициент alpha не берется «на глаз». Он жестко привязан к частоте прибора ($f_s$) и частоте среза шума ($f_c$), которую мы хотим отсечь.
В кардиографах мышечный шум обычно отсекают на частоте выше 35 Гц или 40 Гц. Формула расчета выглядит так:
$$\Delta t = \frac{1}{f_s}$$
$$\tau = \frac{1}{2 \cdot \pi \cdot f_c}$$
$$\alpha = \frac{\Delta t}{\tau + \Delta t}$$
Для Холтера с частотой 250 Гц и фильтрацией шумов выше 35 Гц ваша команда студентов может заранее рассчитать константу alpha. Она будет равна примерно 0.46


------------------------------
## Шаг 2. Как написать «умный» фильтр, который не портит R-пики
Чтобы фильтр низких частот (EMA) сглаживал мышечную дрожь, но не трогал острые вершины ударов сердца, его делают адаптивным.
Логика простая: если график ползет плавно (идет мышечный шум или покой), мы фильтруем его жестко. Но если точка резко взлетает вверх (начинается R-пик), прибор временно отключает сглаживание, чтобы записать вершину удара со стопроцентной точностью.
Вот как выглядит этот продвинутый C++ код:

      #include <iostream>#include <vector>#include <cmath> // Для std::abs
      // Умный адаптивный фильтр низких частотstd::vector<double> removeMuscleNoiseAdaptive(const std::vector<double>& dirtySignal, double defaultAlpha) {
          std::vector<double> cleanSignal(dirtySignal.size());
          cleanSignal[0] = dirtySignal[0];
      
          // Порог резкости. Если разница между точками больше этого значения,
          // значит это не мышечный шум, а начало R-пика (удара сердца).
          const double PEAK_THRESHOLD = 50.0;
      
          for (size_t i = 1; i < dirtySignal.size(); ++i) {
              // Считаем, насколько резко прыгнул сигнал по сравнению с прошлой чистой точкой
              double diff = std::abs(dirtySignal[i] - cleanSignal[i - 1]);
             
              double currentAlpha = defaultAlpha;
      
              // Если это резкий прыжок (удар сердца) — увеличиваем alpha до 1.0 (отключаем фильтр)
              if (diff > PEAK_THRESHOLD) {
                  currentAlpha = 1.0;
              }
      
              // Применяем формулу сглаживания
              cleanSignal[i] = currentAlpha * dirtySignal[i] + (1.0 - currentAlpha) * cleanSignal[i - 1];
          }
      
          return cleanSignal;
      }
      int main() {
          // Симулируем сложную ситуацию:
          // Точки 100-105-96 — это мелкий мышечный шум.
          // Точка 500 — это мгновенный и важный R-пик (удар сердца).
          std::vector<double> signal = {100, 105, 96, 102, 500, 101, 98};
      
          // Запускаем фильтр с базовым alpha = 0.46 (для 250 Гц Холтера)
          std::vector<double> cleanSignal = removeMuscleNoiseAdaptive(signal, 0.46);
      
          std::cout << "Оригинал -> Очищенный сигнал:\n";
          for (size_t i = 0; i < signal.size(); ++i) {
              std::cout << signal[i] << " \t-> \t" << cleanSignal[i] << "\n";
          }
      
          return 0;
      }

## Результат работы алгоритма:

* Мелкая мышечная дрожь 105 -> 96 -> 102 плавно сгладится и превратится в ровную линию около 100.
* Острый пик 500 алгоритм распознает как удар сердца, временно переключит alpha в значение 1.0 и пропустит его без единого искажения амплитуды.
