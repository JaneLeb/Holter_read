### 🔧Архитектурная роль и стратегия для C++ ядра: как обойти конкурентов

Ядро должно выполнять роль высокопроизводительного сервиса первичной обработки (Heavy Lifter):
  
  * Парсинг:
    Суточная запись Холтера — это огромный бинарный файл (в форматах EDF, MIT-BIH или закрытых вендорских форматах вроде Schiller/Incart). C++ парсер мгновенно читает этот массив в оперативную память.
  
    
  * Фильтрация и Детекция (Пан-Томпкинс):
  Код на C++ очищает сигнал от изоэлектрического дрейфа (дыхания) и шумов, а затем находит все R-зубцы (QRS-комплексы). На выходе - точнейшая сетка кардиоинтервалов (RR-интервалы) и карта нарушений ритма.

Алгоритм Пана-Томпкинса (Pan-Tompkins) — это классика кардиоанализа. Он работает на жесткой математике (фильтры, дифференцирование, возведение в квадрат, адаптивные пороги) и идеально подходит для C++, так как требует минимальных ресурсов и работает молниеносно. 

Фильтр опасныех участков согласно типам активностей из дневника (возможно пока достаточно спорта и сна) или другие критически важные на этапе MVP ( например - бег + сердце билось часто = норма, а покой + сердце билось часто = патология).

  * Передача данных для анализа ИИ. Data Science модель должна получать не только отфильтрованные данные, но и контекст (например, отправлять участок ЭКГ с пометкой «пациент сидел, но пульс подскочил».)

## 🐾 Что такое «Пайплайн обработки данных в консоли»?
Разработка всегда начинается с консольного пайплайна (бизнес-логики). Пока код не научится правильно обрабатывать цифры в памяти, делать графический интерфейс (кнопочки и графики) нет смысла.

Пайплайн (Pipeline — конвейер) — это последовательность шагов, через которую проходят данные от начала до конца.
В C++ для проекта это означает программу без графического интерфейса (черное окошко консоли), которая работает по строгой цепочке, шаг за шагом:

##### Пример универсального парсера модульной архитектуры:

-   Шаг 1. Прослойка (Слой абстракции):


  Код не должен работать с конкретным файлом это можно сделать, если создать базовый класс EcgReader, от которого наследуются EdfReader, IshneReader и RawBinaryReader.
   
   
-   Шаг 2. «Сырой импорт» (универсальный бинарный парсер с настраиваемым сдвигом):


  Как это устроено внутри: На самом деле, производители секретных файлов не изобретают квантовую физику. Их файлы — это обычный бинарный поток (массив чисел short или int), перед которым идет «заголовок» (Header) с метаданными (ФИО пациента, частота дискретизации, шаг квантования).
   Если формат файла неизвестен, даем пользователю (инженеру клиники или врачу) возможность импортировать его как RAW Binary (RAW import). Добавим в интерфейс ручные настройки: «Укажите смещение заголовка в байтах, частоту (например, 250 Гц) и разрядность (16 бит)». C++ парсер просто прочитает этот массив байт напрямую. Врачи будут молиться на эту функцию, так как смогут «оживить» данные с любых древних аппаратов или подать на вход импортные данные из засекреченных производителями файлов.
   
   
-   Шаг 3. Локальный реверс-инжиниринг:

  
  На первом этапе для акселератора выберираем 1–2 самых популярных аппарата в тех больницах, с которыми найдем точки соприкосновения (например, «Инкарт» или Schiller). Берем hex-редактор, находим, где в их бинарниках начинаются чистые сигналы ЭКГ, и пишем под них кастомный парсер на C++. Это покажет инвесторам, что стартап способен решать сложные интеграционные задачи.

Программа отбрасывает очевидный мусор (например, если пульс внезапно стал 0 на одну секунду, а потом снова 70 — это отошел провод, а не остановка сердца). Но сердце физически не может сокращаться со скоростью 600 ударов в минуту или мгновенно остановиться на одну секунду, а в следующую выдать пульс 70.
«Ложные тревоги» (Как убрать мусор без высшей математики) - решение на C++: написать алгоритм валидации артефактов (физических помех).


##### 🔎 Представим наш цифровой холтер
Как выглядят входные данные (Что дано)
Забудьте про графики ЭКГ с распечаток А4. Для программы холтер — это просто текстовый файл, где каждую секунду записывается два числа:

   1. Время (секунда от начала записи).
   
   2. Пульс (количество ударов в минуту).

Пример файла: 1.0 72, 2.0 75, 3.0 0 и так далее. За сутки таких строчек будет около 86 000.


Вместо миллионов строк, начнем с файла всего на 10 строк, чтобы визуально понять, как C++ превращает строчки текста в структуру данных. Предположим, что у нас есть текстовый файл holter.txt. В нем датчик холтера записал время (в секундах) и пульс. 
Специально сделаем аномалию: на 4-й секунде пульс улетит в космос (шум), а на 7-й секунде — упадет до нуля (пауза в сердце).

Пишем базовый код на C++


Ниже представлен простой, но рабочий код на C++. Он делает три вещи: открывает этот файл, считывает данные в структуру (память) и применяет те самые правила фильтрации if/else, о которых мы говорили.

    #include <iostream>#include <fstream>#include <vector>
    // 1. Создаем структуру для хранения одной строчки данныхstruct HolterPoint {
        double time;
        int pulse;
        std::string status; // Сюда запишем вывод: "Норма", "Шум" или "Пауза"
    };
    int main() {
        // 2. Открываем файл для чтения
        std::ifstream file("holter.txt");
        if (!file.is_open()) {
            std::cout << "Ошибка: Не удалось открыть файл holter.txt" << std::endl;
            return 1;
        }
    
        std::vector<HolterPoint> data;
        double t;
        int p;
    
        // 3. Читаем файл построчно до самого конца (Парсер)
        while (file >> t >> p) {
            HolterPoint point;
            point.time = t;
            point.pulse = p;
           
            // 4. Логика фильтрации и анализа (Умный движок)
            if (p > 200) {
                point.status = "ШУМ (Проверить датчик)";
            } else if (p == 0) {
                point.status = "КРИТИЧЕСКАЯ ПАУЗА РИТМА";
            } else {
                point.status = "Норма";
            }
    
            // Сохраняем обработанную точку в наш массив (вектор)
            data.push_back(point);
        }
        file.close();
    
        // 5. Выводим результат на экран (Имитация отчета для врача)
        std::cout << "=== ОТЧЕТ ДЛЯ ВРАЧА ===" << std::endl;
        for (const auto& point : data) {
            std::cout << "Время: " << point.time << " сек | "
                      << "Пульс: " << point.pulse << " | "
                      << "Статус: " << point.status << std::endl;
        }
    
        return 0;
    }

------------------------------
 Как это работает внутри компьютера?

 

   1. std::ifstream открывает текстовый файл на жестком диске.
   2. Цикл while (file >> t >> p) бежит по файлу. На каждом шаге он берет первую цифру (время) и кладет в переменную t, а вторую цифру (пульс) — в p.
   3. Конструкция if/else мгновенно классифицирует эту секунду. Она не путает помеху (pulse > 200) с реальной остановкой сердца (pulse == 0).
   4. В конце программа выводит чистый отчет, где проблемные места уже размечены для врача.

##### Что должна делать программа (Три простых правила)


Нужно написать логику, которая проверяет пульс каждую секунду. Переведем слова врачей на язык кода:

   1. Правило «Всё хорошо» (Норма):
   * Что говорит врач: «Синусовый ритм в пределах возрастной нормы».
   * Что это в коде: Если пульс находится в диапазоне от 50 до 140, программа считает, что всё в порядке.
   
   
   2. Правило «Провод отошел» (Помеха/Шум):
   * Что говорит врач: «Артефакты записи, связанные с физической активностью».
   * Что это в коде: Сердце человека физически не может выдать пульс 400 или 500 ударов. Если в файле появилось число больше 200, это не болезнь — это задел провод. Программа должна пометить эту строчку как ШУМ.
   
   
   3. Правило «Остановка сердца» (Пауза ритма):
   * Что говорит врач: «Эпизод асистолии / синоатриальной блокады».
   * Что это в коде: Если пульс внезапно равен 0 или меньше 30 ударов в минуту — это опасный момент. Программа должна пометить эту строчку как КРИТИЧЕСКАЯ ПАУЗА.


##### Медицинские и физиологические фильтры (простая логика и условия if/else):

   * Фильтр «Отошедший провод» (Дрейф изолинии): Если датчик отклеился, значения напряжения улетают в космос. Вы пишете условие: «Если значение напряжения $> X$ милливольт на протяжении более 0.5 секунд, то этот кусок данных мы не считаем аритмией. Мы помечаем его тегом [ПОМЕХА: ОТОШЕЛ ДАТЧИК] и не показываем врачу как тревогу».
   
   
   * Фильтр «Физиологический предел»: Если между двумя ударами сердца прошло менее 0.2 секунд (что соответствует пульсу > 300 уд/мин, что невозможно для живого человека в покое) — это механический шум (человек почесал грудь, задел провод рукой). Код: помечаем этот пик как [ШУМ] и сглаживаем его.
   
   
   * Фильтр «Контекстный анализ»: Если программа видит «остановку сердца» на 2 секунды, ваш код должен проверить, что было до и после. Если до этого пульс был стабилен (70-70-70-0-70-70), то это технический сбой (потеря пакета данных). Если бы это была реальная блокада сердца, пульс бы плавно замедлялся или менялся.


##### ❌ Что НЕ НАДО делать: 

Написание детектора R-пиков с нуля
Алгоритм Пана-Томпкинса в классическом виде уже реализован на C и C++ в десятках вариаций. Писать его математику заново — потеря времени для акселератора.
Не нужно изобретать математику фильтров и вручную писать формулы преобразования Фурье или высчитывать дифференциальные уравнения. Вся базовая фильтрация шумов в C++ сводится к вызову готовых функций из библиотек.


##### Что взять готовое:
   * [PanTompkinsQRS](https://github.com/rafaelmmoreira/PanTompkinsQRS) (чистый ANSI-C) или [Pan-Tompkins-FixedPoint](https://github.com/hooman650/Pan-Tompkins-FixedPoint) (оптимизированный под fixed-point вычисления). Это готовые, компактные файлы (.c и .h), которые инициализируются функцией PT_init() и принимают поток сэмплов в реальном времени.
   
   
   * [Cpp_Ecg_Detectors](https://github.com/Foxpace/Cpp_Ecg_Detectors) (C++) — библиотека, которая содержит С++ версию Пана-Томпкинса и встроенную библиотеку фильтров IIR1 (для фильтров Баттерворта и Чебышева, чтобы убирать шумы).
   
   
   *  [IIR1 (IIR Filter library)](https://github.com/berndporr/iir1) - библиотека на С++. Она создана специально для фильтрации сигналов в реальном времени.
Для очистки ЭКГ от шумов в медицине используют три стандартных фильтра. Код фильтрации в ней выглядит буквально в 3 строчки:

      Iir::Butterworth::LowPass<4> f; // Создали фильтр Баттерворта 4-го порядка
      f.setup(250, 35); // Настроили: частота записи 250 Гц, отсекаем всё выше 35 Гц
      double clean_sample = f.filter(dirty_sample); // На выходе — чистый сигнал!

Просто пропускаем бинарный массив через этот готовый фильтр, и 80% шумов исчезнут сами.

* Убрать дыхание (изолинию): Фильтр высоких частот (High-pass). Он отсекает всё, что колеблется медленнее 0.5 Гц.
  
* Убрать наводку от розетки (50 Гц в РФ): Режекторный фильтр (Notch filter). Он просто «вырезает» ровно частоту 50 Гц из сигнала.
  
* Убрать дрожание мышц: Фильтр низких частот (Low-pass). Он отсекает мелкий высокочастотный шум выше 35 Гц.

##### ⁉️ Точка роста (Что делать надо): 
Классический алгоритм Пана-Томпкинса часто ошибается на «грязных» суточных записях Холтера (когда пациент идет, и электроды трутся об одежду). Нужно взять готовый сишный код и дописать адаптивные пороги фильтрации (логику «умного» отсечения шумов), создав свою модификацию (условно, Pan-Tompkins++).

Вместо того чтобы пытаться идеально отфильтровать 100% шума (что физически невозможно, если провод оторвался), нужно научить свой алгоритм не фильтровать, а распознавать безнадежный шум.
Это называется SQA (Signal Quality Assessment — Оценка качества сигнала):

   1. Считаем среднюю амплитуду (высоту) нормального ЭКГ-сигнала.
   
   
   2. Если в какой-то секунде Пана-Томпкинса начинает «колбасить» и амплитуда прыгает в 5–10 раз выше нормы (пациент почесал грудь) — ваш алгоритм на C++ не пытается это очистить.
   
   
   3. Код просто ставит метку: Участок с 14:02:10 по 14:02:15 — ШУМ, данные недостоверны.
   
   
   4. Врач открывает программу, видит серую полосу вместо графиков и говорит вам спасибо. Почему? Потому что вы не подсунули ему 50 ложных аритмий на этом участке, а честно сказали: «Здесь больной шевелился, пропусти этот отрезок».


Результат для врача: программа отфильтрует 90% этого мусора. Вместо 1000 ложных тревог врач откроет отчет и увидит всего 5–10 реальных аномалий, которые действительно требуют его внимания.

Настоящие данные ЭКГ и Холтера (например, из той самой открытой базы MIT-BIH) обычно хранятся в простейших файлах .csv или .txt.
Внутри такого файла лежит просто бесконечный столбец чисел — показания электрического потенциала сердца, зафиксированные прибором через равные доли секунды.
Вот готовый код класса HolterDataReader. Он открывает файл, читает эти числа «пачками» по 187 штук (размер одного удара сердца) и передает их в массив, который готов к отправке в ИИ [stem-calculative-problem-solving].


## Код парсера ЭКГ-файлов на C++
      
      
      #include <iostream>#include <fstream>#include <vector>#include <string>
      class HolterDataReader {private:
          std::string file_path;
      public:
          // Конструктор: запоминаем путь к файлу с данными Холтера
          HolterDataReader(const std::string& path) : file_path(path) {}
      
          // Функция чтения: вырезает из файла следующий удар сердца (187 чисел)
          std::vector<float> get_next_heartbeat(std::ifstream& file, bool& has_more_data) {
              std::vector<float> heartbeat;
              std::string line;
      
              // Читаем ровно 187 строк (точек) из файла
              while (heartbeat.size() < 187 && std::getline(file, line)) {
                  if (!line.empty()) {
                      // Переводим строчку текста в число типа float и кладем в массив
                      heartbeat.push_back(std::stof(line));
                  }
              }
      
              // Проверяем, не закончился ли файл целиком
              if (file.peek() == EOF) {
                  has_more_data = false;
              } else {
                  has_more_data = true;
              }
      
              // Если файл внезапно оборвался и мы не наскребли 187 точек,
              // добиваем массив нулями, чтобы ИИ не выдал ошибку из-за размера
              if (heartbeat.size() < 187 && !heartbeat.empty()) {
                  heartbeat.resize(187, 0.0f);
              }
      
              return heartbeat;
          }
      };

            return 1;
        }

        std::cout << "[Старт]: Начинаем полный анализ суточного Холтера..." << std::endl;

        }

### Шаг 2. Класс Visualizer (График прямо в консоли!)
Полноценный графический интерфейс (GUI) на С++ требует тяжелых библиотек (типа Qt), что сложно для первого пэт-проекта. Но мы можем сделать псевдографику прямо в консоли с помощью обычных текстовых символов * или #!
Этот класс будет брать 187 точек удара сердца, масштабировать их и «рисовать» график ЭКГ прямо в терминале Школы 21.

      #include <algorithm>
      class Visualizer {public:
          // Рисует один удар сердца в консоли
          void draw_heartbeat(const std::vector<float>& heartbeat_data, int width = 60, int height = 15) {
              if (heartbeat_data.empty()) return;
      
              std::cout << "\n--- График подозрительного удара сердца ---\n";
      
              // Находим минимум и максимум для масштабирования графика по вертикали
              auto [min_it, max_it] = std::minmax_element(heartbeat_data.begin(), heartbeat_data.end());
              float min_val = *min_it;
              float max_val = *max_it;
              float range = max_val - min_val;
              if (range == 0.0f) range = 1.0f;
      
              // Идем сверху вниз по строкам экрана
              for (int r = height; r >= 0; --r) {
                  float current_y = min_val + (static_cast<float>(r) / height) * range;
                 
                  // Идем слева направо по шагам данных
                  for (int c = 0; c < width; ++c) {
                      // Масштабируем индекс массива под ширину экрана
                      int data_idx = (c * heartbeat_data.size()) / width;
                      float val = heartbeat_data[data_idx];
      
                      // Если значение близко к текущей строке — рисуем точку графика
                      if (std::abs(val - current_y) < (range / (height * 2.0f))) {
                          std::cout << "#";
                      } else if (c == 0) {
                          std::cout << "|"; // Ось Y
                      } else {
                          std::cout << " ";
                      }
                  }
                  std::cout << "\n";
              }
              std::cout << std::string(width, '-') << "\n"; // Ось X
          }

      int main() {

## День 9. Пишем инструкцию по установке vcpkg, чтобы подключить ИИ без ошибок или создадим шаблон класса для обработки ошибок, если файл пациента поврежден

В медицине обработка ошибок — это критически важная часть. Если файл поврежден, данные искажены или в них пропущены строки, классическая программа упадет в сегфолт (segmentation fault), а реальный прибор в больнице просто выключится, что недопустимо.
В С++ для этого используется механизм исключений (exceptions). Мы создадим собственный класс ошибки HolterException, а также добавим в наш парсер систему «умной проверки» (валидации) данных на лету


------------------------------
### Шаг 1. Создаем класс медицинской ошибки HolterException
Этот класс наследуется от стандартного std::runtime_error. Он будет собирать информацию о том, в каком именно месте и почему произошел сбой.

    #include <stdexcept>#include <string>
    class HolterException : public std::runtime_error {private:
        std::string error_details;
    public:
        // Конструктор принимает сообщение и контекст (например, имя файла или номер строки)
        HolterException(const std::string& message, const std::string& context)
            : std::runtime_error(message), error_details(context) {}
    
        // Метод для получения подробностей ошибки для логов
        std::string get_details() const {
            return "Контекст ошибки: " + error_details;
        }
    };

------------------------------
### Шаг 2. Добавляем «умную проверку» в парсер HolterDataReader
Теперь перепишем функцию чтения данных. Если в файле окажутся битые данные (например, вместо чисел будет написан текст «ошибка датчика», или числа будут физически невозможными для ЭКГ), наш парсер не упадет, а выбросит исключение.

    #include <iostream>#include <fstream>#include <vector>#include <string>
    class HolterDataReader {private:
        std::string file_path;
        int line_counter = 0; // Считаем строки, чтобы точно знать, где ошибка
    public:
        HolterDataReader(const std::string& path) : file_path(path) {}
    
        std::vector<float> get_next_heartbeat(std::ifstream& file, bool& has_more_data) {
            std::vector<float> heartbeat;
            std::string line;
    
            while (heartbeat.size() < 187 && std::getline(file, line)) {
                line_counter++;
                if (line.empty()) continue;
    
                try {
                    // Пытаемся конвертировать строку в число
                    float val = std::stof(line);
    
                    // ВАЛИДАЦИЯ: Физический контроль данных (по Сатурну)
                    // Вольтаж ЭКГ обычно лежит в пределах от -5.0 до 5.0 мВ.
                    // Если там число 999.0 — это явный сбой оборудования.
                    if (val < -10.0f || val > 10.0f) {
                        throw HolterException(
                            "Критический всплеск напряжения (аномальный вольтаж)",
                            "Файл: " + file_path + ", Строка: " + std::to_string(line_counter) + ", Значение: " + line
                        );
                    }
    
                    heartbeat.push_back(val);
    
                } catch (const std::invalid_argument&) {
                    // Если stof не смог перевести текст в число (например, в файле написано "NULL" или буквы)
                    throw HolterException(
                        "Поврежденный формат данных (невозможно прочитать число)",
                        "Файл: " + file_path + ", Строка: " + std::to_string(line_counter) + ", Текст: '" + line + "'"
                    );
                }
            }
    
            if (file.peek() == EOF) {
                has_more_data = false;
            } else {
                has_more_data = true;
            }
    
            if (heartbeat.size() < 187 && !heartbeat.empty()) {
                heartbeat.resize(187, 0.0f);
            }
    
            return heartbeat;
        }
    };

------------------------------
### Шаг 3. Защищаем главный цикл программы с помощью try - catch
Теперь мы оборачиваем наш главный цикл в блок try-catch. Если файл окажется идеальным — программа отработает как обычно. Если внутри файла обнаружится текстовый мусор или безумные цифры — программа не упадет, а аккуратно запишет ошибку в лог и продолжит работу или штатно завершится.

    int main() {
        try {
            AIAnalyzer holter_ai("mit_bih_arrhythmia.onnx");
            HolterDataReader reader("patient_data.txt");
            ReportSaver saver("final_report.txt");
            Visualizer visualizer;
    
            std::ifstream file("patient_data.txt");
            if (!file.is_open()) {
                // Бросаем системное исключение, если файла вообще нет на диске
                throw std::runtime_error("Файл 'patient_data.txt' не найден в папке проекта!");
            }
    
            bool has_data = true;
            int heartbeat_counter = 1;
    
            std::cout << "[Система]: Сканирование запущено..." << std::endl;
    
            while (has_data) {
                // Метод ридера может выбросить HolterException, если наткнется на битые данные
                std::vector<float> current_beat = reader.get_next_heartbeat(file, has_data);
                if (current_beat.empty()) break;
    
                std::string verdict = holter_ai.analyze_heartbeat(current_beat);
    
                if (verdict.find("ВНИМАНИЕ") != std::string::npos) {
                    std::cout << "\n[!] Патология в ударе №" << heartbeat_counter << std::endl;
                    saver.log_anomaly(heartbeat_counter, verdict);
                    visualizer.draw_heartbeat(current_beat);
                }
    
                heartbeat_counter++;
            }
    
            std::cout << "\n[Успех]: Анализ завершен без критических сбоев." << std::endl;
            file.close();
    
        }
        // Ловим наши кастомные медицинские ошибки
        catch (const HolterException& ex) {
            std::cerr << "\n[КРИТИЧЕСКАЯ ОШИБКА ДАННЫХ]: " << ex.what() << std::endl;
            std::cerr << ex.get_details() << std::endl;
            std::cerr << "[Действие]: Замените кабель Холтера или проверьте исходный файл." << std::endl;
            return 2;
        }
        // Ловим любые другие стандартные ошибки С++ (например, нехватку памяти)
        catch (const std::exception& ex) {
            std::cerr << "\n[СИСТЕМНЫЙ СБОЙ]: " << ex.what() << std::endl;
            return 3;
        }
    
        return 0;
    }

## День 10. Последний шаг, чтобы запустить это на компьютере: посмотрим инструкцию по развертыванию vcpkg, чтобы CMake без проблем подтянул библиотеку ONNX

Чтобы подключить библиотеку машинного обучения ONNX Runtime на вашей рабочей машине с помощью пакетного менеджера vcpkg, вам понадобится выполнить несколько стандартных команд в терминале.
Права администратора часто ограничены, и этот способ идеален — он разворачивает все библиотеки локально, прямо в вашей домашней папке, не требуя системных паролей sudo

.
------------------------------
### Шаг 1. Клонируем и настраиваем vcpkg (если его еще нет)
Откройте терминал и перейдите в вашу домашнюю директорию. Скачайте vcpkg с официального GitHub и запустите его внутренний скрипт сборки:

    cd ~
    git clone https://github.com
    cd vcpkg
    ./bootstrap-vcpkg.sh

После этого в папке vcpkg появится готовый исполняемый файл. Чтобы система всегда знала, где он лежит, добавьте путь к нему в переменную окружения.

    export VCPKG_ROOT=$HOME/vcpkg
    export PATH=$VCPKG_ROOT:$PATH

------------------------------
### Шаг 2. Скачиваем библиотеку ИИ
Теперь, находясь в консоли, дайте команду vcpkg найти, скачать и скомпилировать библиотеку ONNX Runtime специально под вашу операционную систему:

    vcpkg install onnxruntime

Процесс может занять несколько минут, так как vcpkg сам скачает исходный код, проверит зависимости и соберет файлы .a или .dylib, которые мы будем подключать к С++.
------------------------------
### Шаг 3. Собираем наш медицинский проект через CMake
Создайте новую папку для вашего проекта, например HolterProject. Положите туда ваш файл с кодом main.cpp (где прописаны классы ридера, ИИ, визуализатора и обработки ошибок) и созданный ранее файл CMakeLists.txt.
Теперь запустите сборку, явно указав CMake, что нужно использовать менеджер пакетов vcpkg. Для этого передается специальный флаг CMAKE_TOOLCHAIN_FILE:

    #### 1. Заходим в папку с проектом
    cd ~/HolterProject
    #### 2. Создаем отдельную папку для сборки (чтобы не мусорить в коде)
    mkdir build && cd build
    #### 3. Конфигурируем проект (CMake сам подтянет ONNX через vcpkg)
    cmake .. -DCMAKE_TOOLCHAIN_FILE=$HOME/vcpkg/scripts/buildsystems/vcpkg.cmake
    #### 4. Компилируем финальный исполняемый файл
    cmake --build .

------------------------------
### Финал: запуск программы
После успешной компиляции в папке build появится ваш готовый файл. Положите рядом с ним скачанный файл модели mit_bih_arrhythmia.onnx и файл с ЭКГ patient_data.txt, после чего запускайте:

./holter_app

Терминал  превратится в диагностический центр, выдавая логи ошибок, процентные вероятности ИИ и рисуя символьные графики ЭКГ прямо на экране.
Вы полностью развернули рабочую среду и написали каркас серьезного продукта. С этого шага проекты официально превратились в реальные инженерные навыки.
